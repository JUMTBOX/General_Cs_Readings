### 1.5 가상 메모리
- `프로세스의 일부만 메모리에 로드하고, 나머지는 디스크에 둔 상태로 프로세스를 실행하는 방식`
- 프로세스 전체가 메모리에 로드 되지 않아도 실행하는 데 문제가 없다는 점에서 착안

**pros**
- 프로그램이 메모리 크기에 대한 제약을 덜 받을 수 있음
- 동시에 많은 프로그램을 실행하므로 CPU 이용률과 처리율을 높일 수 있음
- 필요한 영역에 메모리에 로드하여 스와핑 횟수를 줄여 프로그램 실행 속도를 높일 수 있음


**요구 페이징**
- `프로세스에서 필요한 페이지만 메모리에 로드하는 방식`
- 초기에 필요한 영역만 로드한 후 다른 영역은 요청이 올 때 메모리에 로드
- 프로그램을 실행하다가 물리 메모리(=RAM)에 필요한 페이지가 없을 때 이를 `페이지 폴트(Page Fault)`라고 한다.
- 페이지 폴트가 발생하면 디스크에서 필요한 페이지를 `스왑 인(swap-in)`한다.
- 페이지에 해당하는 메모리 영역이 물리 메모리에 존재하는지는 `페이지 테이블`로 파악 (존재하면 'v(valid)', 존재하지 않거나 유효하지 않은 주소 값이면 'i(invalid)'값 반환)

**스레싱**
- `동시에 일정 수 이상의 프로그램을 실행했을 때 오히려 CPU 이용률이 떨어지는 상황을 말한다`
- 일정 수 이상으로 다중 프로그래밍을 하면 페이지 폴트(page fault)가 자주 발생하여 CPU 이용률이 떨어진다.
- 예방을 위한 방법으로는 `워킹 세트(working set)`를 설정
- `지역성을 기반으로 자주 사용하는 페이지를 저장해 두는 것을 의미한다.`
- 물리 메모리의 프레임에 고정
