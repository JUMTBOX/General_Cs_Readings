### 1.3 스케줄링 알고리즘
- `CPU 스케줄러(단기 스케줄러)가 준비 큐에 있는 프로세스 중 어떤 프로세스를 실행시킬지 결정하는 데 사용`
- 스케줄링 알고리즘은 비선점형(=nonPreemptive)과 선점형(=preemptive)으로 나뉜다.


**비선점형 스케줄링(non-preemptive scheduling)**
- `실행 중인 프로세스가 종료될때까지 다른 프로세스를 실행할 수 없음을 의미`
- `FCFS (First Come First Served)`
  - 준비 큐에 먼저 들어온 프로세스를 먼저 실행 
- `SJF (Shortest Job First)`
  - 실행 시간이 짦은 프로세스가 우선순위를 가지는 알고리즘, SJN(Shortest Job First)라고도 한다.
  - 평균 대기 시간이 가장 짧지만, 실행 시간이 긴 프로세스는 실행 시간이 짧은 프로세스에 밀려 기아 상태가 될 수 있다.
- `HRRN (Highest Response Ratio Next)`

**선점형 스케줄링(preemptive scheduling)**
- `스케줄러가 실행 중인 프로세스를 중단 시키고 다른 프로세스를 실행할 수 있음을 의미`
- `RR (Round Robin)`
  - 프로세스간 우선 순위가 없다. 모든 프로세스를 순서대로 일정 시간 동안 실행하며, 일정 시간을 초과하면 다른 프로세스를 실행 
- `SRTF(Shortest Remaining Time First)`
  - 준비 큐에서 대기 시간이 가장 짧게 남은 프로세를 우선 수행하는 알고리즘
  - A 프로세스가 실행 중일 때 실행 시간이 더 짧은 B 프로세스가 준비 큐에 들어오면 B 프로세스가 CPU를 차지하게 된다.
  - 평균 대기시간 짧음, 수행 시간이 긴 프로세스는 기아 상태 가능성
- `멀티 레벨(Multi Level)`
  - 준비 큐를 목적에 따라 여러 개로 분리하여 사용하는 알고리즘 
  - 분리한 큐는 각각 우선순위가 있고 각자 다른 스케줄링 알고리즘 적용 가능
  - 여러 개의 큐는 `foreground`큐와 `background`큐로 나뉜다. 
  - `foreground`큐에는 응답 속도가 중요한 프로세스가 들어가고, `background`큐에는 응답 속도보다 성능을 중요시하는 프로세스가 들어간다.
