1.6 캐시 메모리
- `CPU와 메인 메모리(=RAM)간에 데이터 접근 시 속도 차이를 줄이기 위해 사용`
- 캐시 메모리에 어떤 데이터를 저장할지는 `지역성`을 바탕으로 결정한다. 
- `CPU가 자주 참조하는 데이터가 고르게 분포되지 않고 특정 부분에 몰려 있는 것`
- 보통 캐시 미스가 발생하면 가져오는 메모리 주소를 포함하여 64Byte를 로드하기 때문에 공간적으로 인접한게 좋다 (메모리 풀) 

**시간 지역성**
- 최근 참조한 내용을 다시 참조할 가능성이 높다.
**공간 지역성**
- 실제 참조한 주소 근처의 내용을 참조할 가능성이 높다.


**캐시 메모리의 매핑 방식**

- 직접 매핑 (direct mapping)
  - 메인 메모리를 일정 크기로 나누고 각 영역을 캐시 메모리에 매핑하는 방식 
- 연관 매핑 (associative mapping)
  - 메모리 영역을 캐시 메모리에 규칙 없이 매핑하는 방식 (적재는 간단하지만, 캐시에서 필요한 메모리 영역을 찾을 때는 비효율)
- 집합 연관 매핑 (set associative mapping)
  - 직접 매핑과 연관 매핑을 결합, 범용적으로 사용된다.
