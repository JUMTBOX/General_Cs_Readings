### 1.4 메모리 관리 전략

- 연속 메모리 할당
- 페이징 세그먼테이션

**논리 메모리(=가상 메모리)와 물리 메모리**
- 프로세스가 보는 메모리 영역 = `논리 메모리`
- 실제로 사용되는 메모리 영역 = `물리 메모리`
- CPU가 프로세스를 실행하며 보는 주소 = `논리 주소 (= 가상 주소)`
- 실제 메모리에 사용되는 주소 = `물리 주소`
- 논리 주소를 물리 주소로 변환하여 주는 하드웨어 장치 = `메모리 관리 장치 (= MMU, Memory Management Unit)`

**연속 메모리 할당**
- 멀티 프로세스 환경에서 여러 프로세스를 메모리에 연속적으로 로드하는 방법

- `고정 분할 방식`: 메모리 영역을 분할한 뒤 각 영역에 프로세스를 할당 (프로세스 수,크기가 제한되고 단편화 현상 발생)
- `가변 분할 방식`: 할당할 프로세스의 크기에 따라 메모리 공간을 분할, 메모리 할당 알고리즘 이용
  - 최초 적합: 가용 메모리 공간에서 프로세스 크기만큼 비어 있는 메모리 공간을 찾아 차례대로 프로세스를 로드 
  - 최적 적합: 할당하려는 프로세스 크기 이상인 가용 메모리 공간 중에서 가장 작은 공간에 프로세스를 할당 (가용 메모리 공간을 모두 탐색해야함)
  - 최악 적합: 할당하려는 프로세스 크기보다 큰 가용 메모리 공간 중에서 가장 큰 공간에 할당 (역시 가용 메모리 공간을 모두 탐색해야함)
  
- `메모리 압축`: 외부 단편화 (External Fragmentation)를 해결하기 위해 메모리 공간 재배치.. 메모리 집약이라고도 함

**비연속 메모리 할당**
- 프로세스의 메모리 영역을 나누어서 메모리 공간에 저장하는 방법
<br/>
<br/>
- `페이징`: 프로세스의 논리 메모리 영역과 물리 메모리 영역을 각각 일정한 크기의 페이지(page)와 프레임(frame)으로 나눈다.
<br> 이때 페이지와 프레임의 크기는 동일, 페이지와 프레임에는 각각 번호를 할당해 프로세스의 페이지와 메모리 프레임을 매핑한다. (페이지 테이블)
<br> 페이지 테이블은 각 프로세스의 PCB(프로세스 제어 블록)에 저장된다.
    - `계층적 페이징`: 페이지 테이블을 다시 페이지로 나눠 페이지 테이블 자체를 페이징하는 방식, 멀티 페이징이라고도 함
    - `해시 페이지 테이블`: 해시 테이블의 각 항목에 저장된 연결 리스트에 페이지 번호를 해싱한 뒤에 첫 번째 요소와 가상 페이지 번호를 비교하는 방식
    - `역 페이지 테이블`: 프레임을 이용해 페이지를 찾는 방식, 페이지로 프레임을 찾는 기존 방식과 반대

<br/>

- `세그먼테이션`: 프로세스의 메모리 영역을 논리적 단위인 세그먼트로 분할해 메모리를 할당
- 세그먼테이션 테이블을 이용해 논리 주소를 물리 주소로 매핑, 세그먼트 번호를 인덱스로 사용하며 세그먼트별 시작 주소인 base와 세그먼트 길이인 limit을 저장한다.
